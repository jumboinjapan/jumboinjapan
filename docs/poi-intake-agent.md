# Агент приёма POI через Telegram

Задача: в поле, во время маршрута, быстро завести новую точку. Владелец шлёт боту фото таблички, скан буклета или пару строк текста — агент исследует место, собирает факты, пишет описание и создаёт **черновик POI** в Airtable, а затем присылает отчёт в тот же чат.

Реализовано 2026-07-11. Код: `src/lib/poi-intake.ts` (ядро), `src/app/api/telegram/webhook/route.ts` (приём), `src/lib/notifications/telegram.ts` (отправка, загрузка файлов).

## Что агент делает

1. **Распознаёт вход**: текст, фото (берётся самый крупный размер), картинка-документ (скан), **PDF-документ** (с 2026-07-14: буклет/брошюра уходит в модель как `input_file`). Всё уходит в модель одним запросом — текст + изображения + PDF.
2. **Исследует**: OpenAI Responses API с инструментом веб-поиска. Если веб-поиск недоступен на аккаунте — повтор без него (агент работает по присланным данным, всё неподтверждённое честно уходит в «открытые вопросы»).
3. **Проверяет дубли**: сверяет название (RU и EN, с нормализацией регистра/ё/пунктуации) со всеми существующими POI. Намеренно шумит в сторону ложных срабатываний — лучше лишний кандидат в отчёте, чем вторая «Асакуса» в базе.
3а. **Определяет родителя** (с 2026-07-14): если место входит в состав другого объекта — павильон храмового комплекса, работа внутри арт-проекта (Echigo-Tsumari, Naoshima) — исследователь извлекает имя родителя (`parentNameRu/En`), интейк ищет его среди существующих POI и при уверенном совпадении ставит связь в поле `Parent POI`. Если родителя в базе нет — **создаёт родителя-заглушку** (Draft, только имя + пометка) и линкует ребёнка на неё, поэтому порядок отправки не важен: следующие объекты того же проекта найдут заглушку по имени. Заглушку можно заполнить руками или прислать боту отдельно — он увидит её как дубль-кандидата. Комментарий владельца «часть …» / «на территории …» — приоритетный сигнал.
4. **Создаёт черновик**: следующий свободный `POI-000NNN`, поля названия/города/префектуры/категорий/часов/сайта, описания в **Draft**-поля (RU и EN). `Copy Status = Draft`, `Fact Check Status = Todo` — на сайт такая точка не попадает.
5. **Отчитывается**: карточка в Telegram — что создано, какие факты найдены, черновик описания, предупреждение о дублях, список неподтверждённого, источники и прямая ссылка на запись в Airtable.

Ограничения по замыслу:

- Агент **никогда не публикует**. Только черновик, до ручной проверки.
- Категории берутся **только из существующих опций** Airtable (список в `POI_CATEGORIES_RU`). Новые опции select наш токен создавать не вправе, да и плодить синонимы вредно.
- Часы работы и цены модель указывает только при подтверждении источником; иначе поле остаётся пустым, а факт — в «открытых вопросах».
- Описание пишется по канонам сайта: третье лицо, сдержанный тон, без клише, без запрещённых транспортных формулировок.

## Подключение (один раз)

### 1. Создать бота

В Telegram у [@BotFather](https://t.me/BotFather): `/newbot` → имя → username. Он выдаст **токен** вида `1234567890:AA...`.

### 2. Переменные в Vercel

Settings → Environment Variables (галочки **Production** и **Preview**):

| Переменная | Значение |
| --- | --- |
| `TELEGRAM_POI_BOT_TOKEN` | токен нового бота от BotFather |
| `TELEGRAM_WEBHOOK_SECRET` | любая случайная строка, например из `openssl rand -hex 16` |
| `TELEGRAM_OWNER_CHAT_ID` | уже есть — chat id владельца (одинаков для всех ботов) |
| `OPENAI_API_KEY`, `OPENAI_MODEL` | уже есть |
| `AIRTABLE_TOKEN`, `AIRTABLE_BASE_ID` | уже есть |

Обе новые переменные пометить **Sensitive**.

### 3. Указать Telegram, куда слать сообщения

После деплоя выполнить один раз (подставив свои значения):

```bash
curl -X POST "https://api.telegram.org/bot<TELEGRAM_POI_BOT_TOKEN>/setWebhook" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://jumboinjapan.com/api/telegram/webhook",
    "secret_token": "<TELEGRAM_WEBHOOK_SECRET>",
    "allowed_updates": ["message"]
  }'
```

Проверка: `curl "https://api.telegram.org/bot<TOKEN>/getWebhookInfo"` — должен показать url и `pending_update_count: 0`.

Для теста на preview-деплое тот же вызов, но с preview-URL; после теста вернуть боевой адрес (webhook у бота один).

## Безопасность

Эндпоинт публичный (его дёргает Telegram), поэтому:

1. **Секрет вебхука** — Telegram шлёт заголовок `X-Telegram-Bot-Api-Secret-Token`; несовпадение → 401. Без `TELEGRAM_WEBHOOK_SECRET` эндпоинт отвергает всё.
2. **Белый список чатов** — обрабатываются только сообщения с `TELEGRAM_OWNER_CHAT_ID`. Чужие получают 200 и игнорируются (чтобы Telegram не ретраил).
3. Токен бота **не утекает в OpenAI**: изображения скачиваются на сервере и передаются как data-URL, а не ссылкой Telegram (в URL файла зашит токен).

## Тайминг

Telegram ждёт ответ за секунды и ретраит запрос, если его нет. Поэтому webhook отвечает `200` сразу («🔎 Принял. Исследую…»), а исследование (веб-поиск + модель + Airtable, до минуты) выполняется в `after()` — уже после ответа. Дубликатов сообщений не будет.
